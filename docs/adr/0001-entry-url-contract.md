# ADR-0001: Entry-Centric URL Contract

## Status

Accepted

## Context

В системе много типов контента (`dash/widget/dataset/connection`) и много мест, где формируются ссылки.
Локальная сборка URL в каждом компоненте быстро приводит к расхождению роутов и broken deep links.

## Decision

Держать единый контракт построения UI URL от `Entry` через общие URL-конструкторы (`scope/type/key -> route`), а не через ad-hoc строки в компонентах.

## Consequences

1. плюсы: предсказуемая навигация, меньше дублирования, проще миграции URL;
2. минусы: выше ответственность нескольких центральных URL-функций;
3. правило для форка: любые новые entry-ссылки сначала интегрировать в общий URL-конструктор, потом расходовать в UI.
